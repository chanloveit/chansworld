# __Giscus__

백엔드 없이 댓글 기능을 구현하려면 골치 아플 게 뻔하다.  
로그인, 권한, 저장소, 스팸 처리 etc. . .  
블로그 하나 만들겠다고 이 모든 걸 직접 구현하고 싶은 사람은 없을 것.

다른 블로그 만드는 분들도 깃허브의 Giscus 또는 Utterances를 많이 쓰는 것 같다.  
두 앱 모두 공통적으로 깃허브 계정으로 로그인하면 누구나 댓글을 달 수 있도록 제공한다.  
서버도 필요 없고 DB도 필요 없다.  
둘 다 거의 별 차이 없는 것 같다만, giscus를 먼저 접했으므로 이를 골랐다.    
(issu보다는 discussion에 작성되는 게 더 합리적인 것 같기도 하다.)  

giscus 공식 문서에서는 몇 가지 옵션만 선택하면 그에 맞는 HTML 태그를 알아서 만들어 준다.  
그걸 그대로 써도 되고, 원한다면 타입스크립트 컴포넌트로 바꿔서 써도 된다.    
설정도 간단하고, 블로그에 붙이기도 쉽다.  
참으로 간편하지 않은가.

# __다크 모드/라이트 모드__

1장에서 말한 것처럼  
아무런 설계도, 아무런 디자인도 없이 만들다 보면  
CSS 속성이 금방 중구난방이 되어버린다.

어디는 색상이 하드코딩되어 있고  
어디는 변수로 빼놓고  
어디는 아예 기본값을 쓰고 있고…  

이 상태에서 테마 전환을 붙이려니  
도저히 답이 나오지 않았다.

그래서 다크 모드를 붙이기 전에  
색상 관련 CSS를 전부 지우고 다시 시작했다. 하하.  

&nbsp;


Tailwind CSS에서는  
`dark:`라는 접두사를 통해  
다크 모드 전용 스타일을 쉽게 정의할 수 있다.

```ts
<div className="bg-white text-black dark:bg-black dark:text-white">     
```

다크 테마에서만 적용하고 싶은 CSS 속성은  
Tailwind CSS의 dark: 접두사 뒤에 작성하면 된다.  
시스템 설정에 따라 초기에는 다크 테마 또는 라이트 테마가 자동으로 적용되며  
네비게이션 바에 배치한 토글 버튼을 통해  
사용자가 직접 테마를 전환할 수 있도록 했다.  

&nbsp;

테마 전환에는 next-themes 라이브러리를 사용했다.  
이 라이브러리는 현재 테마 상태를 관리해주는 역할을 한다.

현재 테마가 dark라면 light로,  
light라면 dark로 전환하는 것만 처리하면 된다.

나머지-테마 상태의 유지, 초기 테마 결정,  
그리고 서버와 클라이언트 간의 차이로 인한 문제들은  
next-themes가 알아서 처리해준다.
