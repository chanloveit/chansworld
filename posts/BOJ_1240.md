# [[백준 1240 - 노드사이의 거리]](https://www.acmicpc.net/problem/1240)
![백준 1240 - 노드사이의 거리](/images/BOJ_1240/Boj1240.jpg)

## 문제 이해 
`N개의 노드`, `N - 1`개의 간선으로 이루어진 트리가 주어지면 두 노드 사이의 거리(weight의 총합)을 구하면 된다.  

## 핵심 아이디어
BFS/DFS로 시작점 → 목적지 탐색 한 번으로 거리를 계산할 수 있지만(__트리__ 이므로), Floyd-Warshall로 한 번에 모든 정점의 최단 거리를 구하고 문제에서 요구하는 최단거리를 출력할 수도 있다.  
나는 후자를 선택했다. N <= 1000이므로 O(n³)이더라도 2초 안에 충분히 통과 가능하다.

## 소스코드
```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;
const int INF = 10000000;

int main(void)
{
    ios::sync_with_stdio(false);
    cin.tie(NULL);
    cout.tie(NULL);

    int n, m;
    cin >> n >> m;
    vector<vector<int>> v(n + 1, vector<int>(n + 1, INF));
    
    for(int i = 1; i <= n; i++){
        v[i][i] = 0;
    }

    for(int i = 0; i < n - 1; i++){
        int x, y, w;
        cin >> x >> y >> w;
        v[x][y] = w;
        v[y][x] = w;
    }

    for(int k = 1; k <= n; k++){
        for(int i = 1; i <= n; i++){
            for(int j = 1; j <= n; j++){
                v[i][j] = min(v[i][j], v[i][k] + v[k][j]);
            }
        }
    }


    while(m--){
        int x, y;
        cin >> x >> y;
        cout << v[x][y] << '\n';
    }
    return 0;
}
```

## 풀이
시간과 메모리 제한이 널널해서 개인적으로 구현하기 편한 Floyd-Warshall을 사용했다만 조건에 따라 풀이가 달라질 수 있겠다.  
`v[i][j] = min(v[i][j], v[i][k] + v[k][j])`  
모든 정점 쌍을 돌아가며 k를 경유해서 가는 게 더 짧은지 비교한다.  
알고리즘의 작동 방식이 이해하기 어렵다면, 휴먼 g++이 되어 모든 반복문에 따른 계산 과정을 수기로 작성해 보자. (실제 알고리즘 수업 과제였다. 이렇게 한 번 하고 나면 뇌리에 박혀 잊고 싶어도 못 잊을 것이다.)
